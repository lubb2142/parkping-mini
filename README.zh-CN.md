# ParkPing 停车找车助手

一个适合中国大陆使用场景的微信小程序原型，解决“停车后忘记车停在哪”的问题。

[English README](./README.md)

## 当前能力

- 一键记录停车位置
- 支持地下停车场楼层记录
- 支持商场名、停车场名、入口提示、柱号/标识记录
- 支持保存车位区域 / 车位号
- 支持额外记录电梯位置与电梯备注
- 支持拍照保存车位、入口、柱号等现场照片
- 支持微信云开发同步停车记录与照片
- 支持保存常用车牌号，预留车牌反向寻车接入位
- 支持在地图上点选校正车位和电梯坐标
- 支持历史停车记录
- 支持独立“开始找车”流程页
- 支持从任意位置直接调用微信地图导航到车位或电梯

## 为什么先做微信小程序

- 在中国大陆，微信小程序的使用门槛最低，打开即用
- `wx.getLocation`、`map`、`wx.openLocation` 已经能覆盖第一版核心能力
- 不依赖旧车硬件，不需要车机或品牌 App 支持

## 目录说明

- `pages/index`：记录停车、编辑备注、保存照片、同步记录
- `pages/find-car`：找车流程页
- `pages/history`：历史停车记录
- `utils/cloud.js`：云同步与照片上传
- `utils/storage.js`：本地缓存

## 如何运行

1. 打开微信开发者工具
2. 导入项目根目录
3. `appid` 可以先用测试号或你自己的小程序 `AppID`
4. 进入首页后允许定位权限
5. 停车时点击“停车并记录位置”

## 云同步配置

1. 在微信开发者工具中开通云开发
2. 创建集合 `parkingRecords`
3. 给当前小程序配置云环境
4. 照片会通过 `wx.cloud.uploadFile` 上传到云存储
5. 记录会写入云数据库，并在新设备上通过云端拉取

## 当前边界

- 地下停车场 GPS 可能漂移，所以保留了“地图点选校正”
- 商场室内电梯位置无法在所有场馆自动精准识别，所以当前版本支持“手动记录电梯点 + 电梯备注”
- 车牌找车不是通用地图能力，必须依赖停车场自身的车牌识别与反向寻车系统开放接口
- 如果目标商场或停车场没有开放数据接口，就只能记录你自己的停车点，不能自动查出车位
